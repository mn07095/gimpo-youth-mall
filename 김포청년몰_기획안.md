# 김포청년몰 과일판매 홈페이지 기획안

## 1. 프로젝트 목표

- 김포청년몰이라는 지역 기반 브랜드 이미지를 살린 과일 판매 홈페이지를 구축한다.
- 사용자는 메인페이지에서 상품을 탐색하고, 상품 상세페이지에서 옵션을 선택한 뒤 실제로 주문과 결제를 완료할 수 있어야 한다.
- 일반 오픈마켓 복제 느낌이 아니라 `로컬 청년 상인`, `산지 직송`, `신선한 제철 과일`이라는 정체성이 보이도록 설계한다.

## 2. 핵심 사용자 흐름

1. 사용자가 메인페이지에 접속한다.
2. 카테고리 또는 추천 상품을 통해 상품목록 페이지로 이동한다.
3. 상품 카드 클릭 후 상품상세 페이지로 이동한다.
4. 옵션, 중량, 수량을 선택한다.
5. 장바구니에 담거나 바로구매를 선택한다.
6. 주문/결제 페이지에서 배송지와 결제정보를 입력한다.
7. 결제 완료 후 주문완료 페이지를 확인한다.

## 3. 사이트 구조

```text
김포청년몰
├─ 메인페이지
├─ 상품목록 페이지
├─ 상품상세 페이지
├─ 장바구니
├─ 주문 / 결제
├─ 주문완료
├─ 로그인 / 회원가입
├─ 마이페이지
├─ 고객센터
└─ 브랜드 소개
```

## 4. 페이지별 구성안

### 4-1. 메인페이지

목적:
김포청년몰의 첫인상과 대표 상품을 보여주고, 카테고리 및 상세페이지로 자연스럽게 이동시키는 페이지

구성 섹션:

- 상단 헤더
  - 로고
  - 검색창
  - 로그인 / 회원가입
  - 장바구니
- 메인 비주얼 배너
  - 계절 대표 과일 이미지
  - "김포 청년이 엄선한 신선한 과일" 같은 핵심 문구
  - 바로가기 버튼
- 카테고리 퀵 메뉴
  - 사과
  - 배
  - 감귤
  - 토마토
  - 제철과일
  - 선물세트
- 베스트 상품 섹션
  - 많이 판매되는 상품 카드 4~8개
- 오늘의 제철 과일 섹션
  - 시즌성 강조
- 청년 상인 소개 섹션
  - 짧은 소개 문구와 이미지
- 김포청년몰 신뢰 요소
  - 산지 직송
  - 신선 배송
  - 품질 검수
  - 안전 결제
- 공지 / 배송 안내
- 푸터
  - 사업자 정보
  - 고객센터
  - 교환/환불 정책

### 4-2. 상품목록 페이지

목적:
사용자가 원하는 과일을 비교하고 선택할 수 있도록 상품을 정렬하고 탐색하게 하는 페이지

구성 요소:

- 카테고리 제목
- 카테고리 설명 문구
- 정렬 옵션
  - 인기순
  - 최신순
  - 낮은 가격순
  - 높은 가격순
- 필터
  - 할인상품
  - 무료배송
  - 품절 제외
- 상품 카드
  - 상품 이미지
  - 상품명
  - 원산지
  - 할인율
  - 판매가
  - 리뷰 수
  - 찜 버튼

### 4-3. 상품상세 페이지

목적:
실제 구매 전환이 일어나는 핵심 페이지

상단 구성:

- 대표 상품 이미지 갤러리
- 상품명
- 별점 / 리뷰 수
- 정가 / 할인가
- 배송비 및 지역 추가 배송비 안내
- 원산지 / 중량 / 보관방법 / 출고일 안내

구매 영역:

- 옵션 선택 드롭다운
  - 예: 3kg / 5kg / 10kg
  - 예: 가정용 / 선물용
- 수량 선택
- 선택 상품 요약
- 총 결제금액 자동 계산
- 버튼
  - 장바구니
  - 바로구매
  - 찜하기

하단 탭 구성:

- 상세정보
- 리뷰
- Q&A
- 배송 / 교환 / 환불 안내

추가 섹션:

- 추천 상품
- 함께 구매하면 좋은 상품

### 4-4. 장바구니 페이지

구성 요소:

- 선택한 상품 목록
- 옵션 / 수량 수정
- 상품 삭제
- 주문금액 합계
- 배송비 계산
- 최종 결제 예정 금액
- 주문하기 버튼

### 4-5. 주문 / 결제 페이지

구성 요소:

- 주문자 정보
- 수령인 정보
- 배송지 입력
- 배송 요청사항
- 주문 상품 요약
- 상품금액 / 배송비 / 할인 / 최종 금액
- 결제수단 선택
- 결제 동의
- 결제하기 버튼

### 4-6. 주문완료 페이지

구성 요소:

- 주문 완료 메시지
- 주문번호
- 결제금액
- 배송지 요약
- 마이페이지 이동
- 쇼핑 계속하기 버튼

### 4-7. 마이페이지

구성 요소:

- 주문내역
- 배송조회
- 찜한 상품
- 회원정보 수정
- 최근 본 상품

### 4-8. 고객센터

구성 요소:

- 공지사항
- 자주 묻는 질문
- 1:1 문의
- 배송안내
- 교환 / 환불 정책

### 4-9. 브랜드 소개 페이지

구성 요소:

- 김포청년몰 소개
- 청년 상인 소개
- 산지와 공급 과정 소개
- 품질 관리 기준
- 지역 상생 메시지

## 6. 상품 데이터 설계 초안

상품마다 아래 정보가 필요하다.

- 상품 ID
- 카테고리
- 상품명
- 짧은 설명
- 상세 설명
- 정가
- 판매가
- 할인율
- 원산지
- 대표 이미지
- 상세 이미지들
- 옵션 목록
- 재고 수량
- 배송비
- 추가 배송비
- 판매 상태
- 리뷰 평점
- 리뷰 수

옵션 예시:

- 중량: 2kg / 3kg / 5kg / 10kg
- 품질 등급: 가정용 / 선물용
- 포장 방식: 일반포장 / 선물포장

## 7. 디자인 방향

### 브랜드 키워드

- 김포 로컬
- 청년 상인
- 신선함
- 따뜻함
- 믿을 수 있는 직거래

### 디자인 톤

- 시장 느낌은 살리되 촌스럽지 않게 정돈된 스타일
- 대형 오픈마켓보다 따뜻하고 스토리가 보이는 레이아웃
- 과일 이미지가 돋보이도록 배경은 너무 강하지 않게 구성

### 컬러 방향

- 메인 배경: 크림, 아이보리, 밝은 베이지
- 포인트 컬러: 사과 레드, 감귤 오렌지, 잎사귀 그린
- 텍스트 컬러: 짙은 브라운 또는 차콜

### 비주얼 요소

- 과일 사진을 크게 활용
- 둥근 카드형 레이아웃 일부 사용
- 지역성과 청년몰 느낌이 나는 소개 배너 삽입
- 배송, 신선도, 직송 같은 신뢰 아이콘 구성

## 8. 벤치마킹에서 가져올 요소

예시 페이지에서 참고할 요소:

- 상품 상세 상단의 구매영역 배치
- 옵션 선택 드롭다운
- 장바구니 / 바로구매 버튼 구조
- 상세정보 / 리뷰 / Q&A 탭
- 가격, 할인율, 배송정보를 한눈에 보이게 하는 정보 구조
- 추천상품 또는 연관상품 영역

그대로 복제하지 않고 바꿔야 할 부분:

- 헤더와 메인 비주얼
- 브랜드 메시지
- 색상 체계
- 카드 디자인
- 지역성과 청년몰 스토리 영역

## 9. 실제 주문 가능 사이트를 위한 기술 구성 추천

혼자 진행하는 프로젝트 기준으로는 아래 구성이 가장 현실적이다.

### 추천 구성

- 프론트엔드: Next.js
- 스타일링: Tailwind CSS 또는 CSS Module
- 백엔드: Next.js 서버 기능 또는 별도 API
- 데이터베이스: Supabase 또는 PostgreSQL
- 이미지 저장: Supabase Storage 또는 클라우드 스토리지
- 로그인: Supabase Auth 또는 NextAuth 계열
- 결제: 국내 결제 모듈 연동

### 이유

- 한 프로젝트 안에서 화면과 서버 기능을 함께 관리하기 쉽다.
- 상품, 회원, 주문 데이터를 DB로 연결하기 좋다.
- 이후 관리자 페이지도 확장하기 쉽다.
